# Функционал и возможности приложения Throne (Форк Nekoray)

Данный документ содержит полное описание архитектуры, функционала и возможностей desktop-приложения [Throne](https://github.com/throneproj/Throne). Приложение является кроссплатформенным GUI-клиентом для маршрутизации трафика и обхода блокировок.

## 1. Базовый функционал и архитектура
* **Кроссплатформенный GUI-клиент:** Графический интерфейс, разработанный для управления и маршрутизации сетевого трафика через различные прокси-протоколы.
* **Ядро (Core Engine):** Работает на базе **Sing-box** (универсальной высокопроизводительной прокси-платформы), что обеспечивает поддержку самых современных протоколов обхода блокировок.
* **Фреймворк интерфейса:** Написан на **Qt** (C++), что дает нативный, отзывчивый опыт использования и консистентный дизайн на разных операционных системах.

## 2. Поддерживаемые операционные системы
* **Windows:** 
  * Поддержка современных версий (Windows 10, 11) "из коробки".
  * Отдельная ветка для поддержки устаревших версий (Windows 7, 8, 8.1) — ветка `nekoray-win7`.
* **Linux:** Полная поддержка, включая глубокую системную интеграцию. Планируется нативная дистрибуция через AUR, PKGCONFIG и другие пакетные менеджеры.
* **macOS:** Поддержка платформ Apple (требует ручных действий по снятию карантина (quarantine) и выдачи прав Full Disk Access для Терминала для работы повышения привилегий).

## 3. Сетевые протоколы и обход блокировок
Приложение поддерживает огромный спектр современных протоколов:
* **Стандартные прокси:** SOCKS (SOCKS4/SOCKS5), HTTP, HTTPS.
* **Современные защищенные протоколы (VPN/Proxy):**
  * Shadowsocks
  * Trojan
  * VMess / VLESS
  * TUIC
  * Hysteria / Hysteria2
  * Wireguard
  * SSH

## 4. Продвинутая маршрутизация трафика
* **Цепочки подключений (Chaining Outbounds):** Возможность каскадного соединения (multi-hop). Позволяет направлять трафик через несколько прокси-серверов последовательно для повышения анонимности.
* **Кастомные исходящие соединения (Custom Outbounds):** Создание узкоспециализированных точек выхода трафика.
* **Пользовательские конфигурации (Custom Config):** Полностью ручная настройка маршрутизации и конфигураций Sing-box для продвинутых пользователей.
* **Дополнительные ядра (Extra Core):** Поддержка интеграции и использования дополнительных прокси-ядер (помимо встроенного Sing-box), что дает возможность расширять поддержку протоколов.

## 5. Управление подписками и профилями конфигураций
* **Поддержка различных форматов подписок (Subscriptions):**
  * Стандартные Share-ссылки (URL-схемы протоколов).
  * JSON-массивы конфигураций исходящих соединений (outbounds).
  * Формат ссылок v2rayN.
  * Ограниченная совместимость с форматами конфигураций **Clash** и **Shadowsocks**.
* **Портативность:** Возможность распространения и использования в виде портативных ZIP-архивов, которые не требуют установки в систему и хранят настройки локально.

## 6. Системные возможности и интеграция
* **Режим TUN-интерфейса (System-wide VPN):** Приложение умеет создавать виртуальный сетевой адаптер (TUN) для прозрачного перенаправления **абсолютно всего** трафика операционной системы через выбранный прокси-протокол (требует прав администратора/root).
* **Управление системным DNS:** Функция перехвата и изменения настроек DNS на уровне ОС для предотвращения утечек DNS-запросов (DNS Leaks) и обхода подмены DNS провайдером.
* **Повышение привилегий (Privilege Escalation):**
  * **Linux:** Использование SUID-битов или флагов `Cap_xxx_admin` (capabilities) для бесшовного управления сетевыми интерфейсами без постоянного ввода пароля.
  * **macOS:** Встроенные механизмы повышения привилегий для применения системных изменений.
  * **Ручной контроль:** Пользователи могут отключить автоматическое повышение привилегий в базовых настройках, если предпочитают выдавать права вручную.

## 7. Пользовательский интерфейс (UI / UX)
* **Встроенный апдейтер (Auto-Updater):** Автоматическая проверка, скачивание и прозрачная замена старых бинарных файлов на новые релизы.
* **Интеграция с QR-кодами:** Генерация QR-кодов для экспорта профилей и сканирование QR-кодов для быстрого добавления новых конфигураций.
* **Глобальные горячие клавиши (Global Hotkeys):** Поддержка системных шорткатов для быстрого управления приложением (например, сворачивание в трей, быстрое включение/отключение TUN) без необходимости открывать окно приложения.
* **Продвинутая работа с конфигурациями:** Поддержка форматов YAML и Protobuf для обработки сложных конфигурационных файлов и быстрой сериализации данных.

## 8. Архитектурные особенности (в сравнении с оригинальным Nekoray)
* **Оптимизация кодовой базы:** Удалены устаревшие и неиспользуемые функции.
* **Упрощенный UI:** Интерфейс стал более чистым и понятным, избавлен от перегруженности оригинального проекта.
* **Современность:** Акцент сделан на использовании Sing-box как основного и наиболее актуального движка для работы с новыми протоколами (Hysteria2, TUIC и т.д.).
