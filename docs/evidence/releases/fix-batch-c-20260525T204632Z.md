## OTP Delivery Contract

По текущему коду task-worker:

- production initial OTP: основной путь через Resend;
- production resend OTP: Brevo используется только если он настроен в runtime env, иначе fallback идёт через Resend;
- local/dev OTP: SMTP/Mailpit path;
- activation link строится из `MAGIC_LINK_BASE_URL`, locale, email и OTP code;
- секреты/API keys в evidence не выводятся.

Production smoke после deploy должен проверить фактическую доставку письма без публикации ключей.
