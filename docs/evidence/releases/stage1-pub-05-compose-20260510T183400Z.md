## Runtime Caveats Before Starting Containers

- The compose references immutable image tags through `CYBERVPN_IMAGE_REGISTRY` and `CYBERVPN_IMAGE_TAG`; Stage 1 live deploy must use pushed images or locally loaded immutable artifacts, not floating `main`.
- Server env files still contain `__REPLACE_BEFORE_LIVE_*` placeholders from STAGE1-PUB-04. Real secrets must be inserted before starting production traffic.
- `DATABASE_URL` and `REDIS_URL` are supplied through the root-only compose `.env` for the no-cost local-data beta path, so those values are not exposed in Git or evidence.
- The task worker intentionally fails closed in production unless a real `CRYPTOBOT_TOKEN` is present; this must be satisfied before payment/reconciliation jobs are enabled.
- `cybervpn-postgres` and `cybervpn-valkey` are available only through the `local-data` profile. They are a cost-saving beta option, not the final managed-data topology from DEC-S1-004/005.
