# Northstar Implementation Security Gates

## Mandatory Before Alpha

- Signed manifest verification exists and is required before activation.
- Strict JWT/JWKS verification exists for gateway admission.
- Session tokens are short-lived and validated before expensive gateway allocation where practical.
- Bootstrap, device registration, token exchange, and webhook flows have replay-aware abstractions, and verified webhook ingress now rejects duplicate deliveries in the bridge domain with durable file-backed and SQLite-backed replay store paths.
- Public bridge error surfaces are structured and redacted.
- Token, bootstrap, refresh, and raw HWID values are redacted in logs.
- The gateway has bounded pre-auth state, session quotas, visible admission failures, and a real localhost QUIC/H3 hello plus bounded relay-data path that still keeps pre-auth scope narrow.

## Mandatory For This Baseline

- Separate manifest-signing and token-signing key roles in code structure.
- Reject unsigned or unverifiable session tokens.
- Require JWT `typ = JWT` and reject duplicate `kid` entries in JWKS inputs.
- Reject expired or not-yet-valid tokens with bounded skew.
- Reject revoked subjects and stale token policy epochs when bridge policy says they are no longer acceptable.
- Reject malformed frames and reserved bits without panics or lossy parsing.
- Reject invalid or non-canonical UDP metadata counts across flow-open, flow-ok, stream-open, and stream-accept frames instead of silently truncating or widening metadata-bearing state.
- Reject invalid or non-canonical UDP metadata lengths across flow-open, flow-ok, stream-open, and stream-accept frames instead of silently truncating, partially decoding, or widening metadata-bearing state.
- Reject invalid or non-canonical UDP metadata type varints across flow-open, flow-ok, stream-open, and stream-accept frames instead of silently remapping, widening, or skipping metadata-bearing state.
- Reject empty-domain or zero-port target addresses and invalid UTF-8 target/message payloads as malformed validation failures instead of letting transport-bearing frames silently accept unusable or ambiguous inputs.
- Reject invalid or non-overlapping session version ranges before admission succeeds.
- Derive client carrier configuration from verified signed carrier-profile data instead of local defaults.
- Preserve signed H3 carrier-profile headers, backoff, and `zero_rtt_policy` in launch planning while keeping effective early data disabled.
- Reject signed H3 request headers that are pseudo-headers, hop-by-hop headers, `host`, `content-length`, `expect`, or otherwise invalid for the centralized H3 request-builder surface.
- Enforce explicit gateway pre-auth budgets before session admission: bounded pending hellos, bounded control-body size, bounded hello wall-clock time, and overload rejection.
- Enforce explicit gateway relay budgets after session admission: bounded accepted-relay concurrency, bounded per-stream raw prebuffer, bounded relay idle time, and overload rejection.

- `ns-auth`: claim validation, Ed25519 key parsing, JWKS resolution, duplicate-`kid` rejection, subject revocation, and stale-policy hooks.
- `ns-manifest`: signature hooks, frozen-schema validation, canonicalization notes, and endpoint/profile linkage.
- `ns-wire`: bounds checks, error mapping, unknown and reserved value handling.
- `ns-storage`, `ns-bridge-api`, `ns-bridge-domain`, and `ns-remnawave-adapter`: refresh credential storage shape, backend boundary discipline, SQLite WAL/busy-timeout behavior, authenticated HTTP-backed service-store health behavior, HTTP route body limits, durable replay-safe webhook ingress, device-policy enforcement, upstream normalization, and schema-drift handling on both remote-store and Remnawave adapter paths.

- `ns-session`, `ns-client-runtime`, `ns-carrier-h3`, and `ns-gateway-runtime`: post-`SERVER_HELLO` contract validation, datagram negotiation, UDP flow-open admission, frozen `UDP_DATAGRAM_UNAVAILABLE` mapping, associated H3 datagram validation, fallback-to-stream behavior, unknown-flow or wrong-mode rejection, and bounded UDP payload, burst, or queue guards.
