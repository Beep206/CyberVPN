# Verification Commands

This file lists the commands Verta expects to use as the repository grows.
Statuses mean:

- `active`: the command or wrapper exists now
- `planned`: expected once the Rust workspace exists
- `placeholder`: intentionally reserved until crate names or tooling choices are finalized

Legacy naming note:

- `Verta` is the canonical public protocol name.
- Commands, env vars, workflow filenames, crate names, and artifact paths still use legacy lowercase `verta` technical identifiers during the migration window.
- Maintained wrappers now accept `VERTA_*` as the canonical alias layer, mirror those values onto legacy `VERTA_*` inputs, and fail closed if both names are set to conflicting values.
- Canonical wrapper outputs now prefer `target/verta/`, while maintained legacy mirrors under `target/verta/` remain readable for existing release evidence and workflow consumers.
