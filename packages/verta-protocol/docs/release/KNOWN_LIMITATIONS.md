# Known Limitations

The items below are accepted, explicit, and non-blocking for Verta `v0.1`.

## Accepted Non-Blocking Limitations

- Only the first carrier implementation, `ns-carrier-h3`, is supported in `v0.1`.
- `0-RTT` remains intentionally disabled by the current normative rules.
- `ns-client-inbound` is still placeholder-only and remains outside the production-critical path.
- The multi-host release-evidence chain is sustained by workflow definitions and repository rules, while the maintained local closure path re-verifies the local supported staging environment.

## Not Accepted As Hidden Debt

- blocking security issues
- hidden support-matrix drift
- dirty-worktree release decisions
- silent widening of the Remnawave boundary
